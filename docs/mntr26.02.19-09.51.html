mntr26.02.19-14.05

System Monitor dapat di-package menjadi aplikasi standalone dalam format .deb, .tar.xz, dan .exe

üöÄ Quick Start

# Install packaging tools (Linux)
sudo apt install dpkg-dev tar xz-utils

# Install PyInstaller (for .exe)
pip3 install pyinstaller

# Create package
python3 package.py [deb|tar|exe|all]

üì¶ Package Types

.deb Package

For: Debian/Ubuntu

Create:

python3 package.py deb

Install:

sudo dpkg -i system-monitor_26.02.19-14.05.deb
monitor

Features:

  • Auto-installs dependencies
  • Global command monitor
  • System integration
  • Easy uninstall

.tar.xz Package

For: Linux Universal

Create:

python3 package.py tar

Install:

tar -xJf system-monitor-26.02.19-14.05.tar.xz
cd system-monitor-26.02.19-14.05
./install.sh
./monitor

Features:

  • Portable
  • No root required
  • Works on any Linux
  • Small size (~3MB)

.exe Package

For: Windows

Create:

python3 package.py exe

Run:

Double-click SystemMonitor-26.02.19-14.05.exe

Features:

  • Single executable
  • No Python required
  • Standalone
  • Portable

üìã Requirements

For .deb packaging:

sudo apt install dpkg-dev

For .tar.xz packaging:

sudo apt install tar xz-utils

For .exe packaging:

pip3 install pyinstaller

üéØ Create All Packages

python3 package.py all

Creates all three package types at once.

üìä Package Sizes

  • .deb: ~5 MB
  • .tar.xz: ~3 MB (compressed)
  • .exe: ~50-100 MB (includes Python runtime)

üîß Package Contents

All packages include:

  • Main application (app.py)
  • All modules (modules/, utils/)
  • Templates and static files
  • Documentation (docs/)
  • Requirements file
  • Launcher script

üêõ Troubleshooting

.deb installation fails

sudo apt --fix-broken install

.tar.xz missing dependencies

pip3 install -r requirements.txt

.exe antivirus warning

Add exception in Windows Defender. The .exe is safe, it's just bundled Python.
